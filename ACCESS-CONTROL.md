# 分权限知识库方案

当前系统分成两个入口：

- 外部免登录入口：只用于公开资料，只同步“需求文档”知识库。
- 管理员/登录入口：用于内部资料、敏感资料、按人员分配账号和权限。

管理员分配模型和自定义角色的具体命令见：[MODEL-ROLE-MANAGEMENT.md](./MODEL-ROLE-MANAGEMENT.md)。

## 公开资料

可以给所有外部访客看的资料，放到“需求文档”知识库。

自动同步服务会每 60 秒检查一次“需求文档”知识库。只有已经加入“需求文档”的文件，才会同步到外部免登录入口。

查看公开知识库当前文件：

```bash
./list-knowledge-files.sh 需求文档
```

如果某个文件不应该公开，可以从“需求文档”知识库移除，但不会删除原始上传文件：

```bash
./unpublish-file-from-public.sh 文件名关键词
```

例如：

```bash
./unpublish-file-from-public.sh G3
```

## 敏感资料

敏感资料不要加入“需求文档”知识库。

建议在管理员后台单独建知识库，例如：

- 内部资料
- 客户A资料
- 售后资料
- 产品规格书

然后为每个知识库创建对应模型，例如：

- 内部资料问答
- 客户A资料问答
- 售后资料问答

模型的系统提示词继续使用“只根据当前绑定知识库回答”的规则。

## 人员账号

管理员后台用于创建和管理账号：

```text
Admin Panel / 管理员面板 -> Users / 用户
```

推荐流程：

1. 管理员创建或批准用户账号。
2. 给用户分组，例如“公开访客”“内部员工”“客户A”。
3. 给不同知识库或模型分配对应用户/分组访问权限。
4. 用户通过管理员/登录入口访问，需要账号密码。

## 重要规则

- 免登录外部链接只放公开资料。
- 敏感资料必须放到单独知识库，并使用登录入口访问。
- 不要把敏感资料加入“需求文档”知识库。
- 当前自动同步是安全模式，不会自动公开所有上传文件。
- 管理员后台更新“需求文档”公开知识库或公开模型后，外部访客端会自动同步显示。
