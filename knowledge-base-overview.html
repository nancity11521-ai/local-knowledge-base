完整功能说明

系统以“管理员维护资料—公开端安全回答—数据反馈改进资料”为主线。以下为当前知识库的全部核心功能。

①

知识库与文件管理

管理员可上传、查看和管理 PDF、Word、Excel、TXT、Markdown 等资料，并将文件绑定到对应知识库。

②

公开问答页面

客户无需注册或登录即可访问公开页面，直接提交产品咨询、规格查询、安装指导或故障现象。

③

限定公开资料范围

公开端只读取指定的 g3问题库。内部文件、测试资料和敏感资料不会被同步到外部。

④

固定客服模型

公开端锁定 requirement-docs-kb 模型，不向访客展示模型选择、后台菜单、搜索、笔记或工作区功能。

⑤

多语言界面与回答

支持中文、英文、日文、韩文、西班牙文、法文、德文、阿拉伯文；访客选择语言后，界面和回答均按该语言显示。

⑥

临时访客对话隔离

公开端采用临时对话策略。访客只应看到自己的当前会话，不能浏览、收到通知或查看其他访客的问题。

⑦

后台与公开端同步

同步公开知识库、绑定文件、原始上传文件、向量库、模型参数、系统提示词和 RAG 检索配置。

⑧

回答一致性控制

后台模型是公开模型的唯一配置来源；温度、知识库范围和检索参数同步，确保两端事实和结论一致。

⑨

Token 缓存

相同请求优先返回缓存结果，减少重复调用模型 API 的 Token 消耗；知识或模型更新后自动清理旧答案缓存。

⑩

访问统计

记录公开页面的访问次数、独立访客、咨询量和咨询语言，供管理员了解服务使用情况。

⑪

问题分析后台

聚合高频问题、问题分类、型号关键词、语言分布和趋势，帮助发现资料缺口与产品服务问题。

⑫

安全运维与恢复

管理员、公开端、缓存统计服务独立运行；支持健康检查、日志排查、手动同步与自动同步，降低单点故障影响。

管理员如何维护资料

1

上传与整理

在管理后台上传 PDF、Word、Excel、Markdown、TXT 等资料，并检查文件内容与名称。

2

绑定公开知识库

需要面向客户公开的文件，必须绑定到 g3问题库；未绑定的文件不会同步到公开端。

3

验证回答

在后台和公开端使用同一问题测试，确认检索资料、事实结论与回答范围一致。

客户如何使用

1

选择语言

在公开页面右上角选择所需语言，系统会按该语言展示界面并生成回答。

2

描述产品与问题

建议包含型号、系统版本、故障现象和已尝试的操作，能提升检索准确度。

3

获取解决建议

系统基于公开资料回答；资料不足时会明确说明，并建议联系官方售后进一步确认。

公开知识库同步原则

唯一来源管理员后台的模型配置、系统提示词、知识库绑定与 RAG 检索配置，是公开端同步的唯一来源。
同步内容公开知识库集合、已绑定文件、上传原件、向量数据库、公开模型配置和检索参数。
回答一致性相同语言、相同问题下，公开端与后台应基于相同资料得出相同事实与结论；不同语言可能存在翻译表达差异。
缓存策略相同请求优先命中 Token 缓存，以降低模型调用成本；资料或配置同步后会自动清除旧回答缓存。
API 配置同步公开模型使用与后台一致的兼容 OpenAI API 配置。管理员更换供应商或 Key 后，应通过同步流程将公开端配置一并更新。
公开端为临时访客对话设计。访客问题会用于匿名统计与问题分析，但访客不应看到其他用户的历史对话。

管理分析能力

后台提供数据化视图，帮助团队把客户问题转化为可改进的知识库与产品服务内容。
